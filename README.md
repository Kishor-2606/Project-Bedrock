## About

I completed a 3-month AI/ML internship, built a production-style application, and cleared multiple in-person technical evaluations. I could use the tools — Python libraries, embeddings, REST APIs, model training pipelines.

What I couldn't always answer was *why* they worked. Why cosine similarity over Euclidean distance. Why vectors get normalized before comparison. What a dot product is actually measuring when it's called "semantic similarity."

Project Bedrock is a structured attempt to close that gap: rebuilding the fundamentals underneath the tools, one week and one project at a time, in the open.

**This is not** a "become an AI engineer in 12 weeks" claim — it's a documented foundation-building process, including the parts that don't work on the first try.

## Roadmap

| Week | Focus | Deliverable | Status |
|:---:|---|---|:---:|
| 01 | Python foundations, OOP basics, exception handling | Standalone script/CLI project | ⬜ |
| 02 | API fundamentals — REST, FastAPI, Pydantic | API-powered application | ⬜ |
| 03 | SQL (MySQL) | Schema design, queries, joins | ⬜ |
| 04 | End-to-end integration | App wired to MySQL | ⬜ |
| 05 | Math foundations — linear algebra, statistics, calculus | Worked problem sets | ⬜ |
| 06 | NumPy & Pandas | Data analysis + visualization | ⬜ |
| 07 | ML core concepts | Preprocessing pipeline + notes | ⬜ |
| 08 | Evaluation metrics | Metrics implemented from scratch + sklearn | ⬜ |
| 09 | ML Project — Regression | Predicting exam scores from study hours | ⬜ |
| 10–11 | ML Project — Classification | Text classifier (TF-IDF / CountVectorizer) | ⬜ |
| 12 | ML Project — Clustering | Customer segmentation (K-Means) | ⬜ |

*Following phase: Deep Learning (neural nets, CNN/RNN) and Generative AI (embeddings, vector databases, RAG).*
